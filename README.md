# Messenger App

A premium‑style chat messenger built with:

- **Backend** – Django REST Framework, MongoEngine, Python 3.11  
  - Service layer (`backend/chat/services.py`) for business logic.  
  - OpenAPI/Swagger documentation via **drf‑spectacular** (`/api/schema/`).

- **Frontend** – React Native (Expo) with TypeScript  
  - Dark‑mode UI, glass‑morphism style, smooth micro‑animations.  
  - Custom hook `useGroupHeader` to keep the navigation header in sync with group participants.  
  - Pagination support for message history (`GET /messages/<id>/?page=1&size=50`).  

## Scripts

| Command | Description |
|---------|-------------|
| `npm install` | Install JS dependencies |
| `npm run lint -- --fix` | Run ESLint + Prettier (strict config) and auto‑fix issues |
| `npm run start` | Start Expo development server |
| `python manage.py runserver` | Start Django backend |
| `python manage.py makemigrations && python manage.py migrate` | DB migrations |
| `python manage.py spectacular --file schema.yml` | Export OpenAPI spec |

## API Overview (generated by drf‑spectacular)

- `POST /register/` – Register a user.  
- `GET /conversations/` – List user conversations (supports pagination).  
- `POST /conversations/` – Create a direct conversation.  
- `POST /groups/` – Create a group (delegates to `services.create_group_service`).  
- `GET /messages/<conversation_id>/` – Paginated message list (`page`, `size`).  
- `POST /messages/<id>/delete/` – Delete a message (for self or everyone).  
- `POST /messages/<id>/seen/` – Mark a message as seen.

## Development notes

- All imports are placed at the top of each file, using the `import { … } from 'module'` style as required.  
- TypeScript interfaces are defined in `src/types.d.ts` for strong typing.  
- Logging is configured via Python’s `logging` module (`logger = logging.getLogger(__name__)`).  
- ESLint configuration (`.eslintrc.js`) includes `eslint-plugin-react`, `eslint-plugin-react-native`, and `@typescript-eslint`.
